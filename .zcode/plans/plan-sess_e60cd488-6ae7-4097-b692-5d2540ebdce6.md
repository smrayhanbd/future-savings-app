# Member Portal Upgrade Plan

## Overview
Upgrade the Member Portal from a basic 4-page top-nav layout into a professional-grade, sidebar-driven experience with **3 new feature areas** (Loans, Requests tracker, Notifications) plus deep UI polish across all existing pages. **Zero Prisma schema changes required** — every feature reuses existing models and server actions.

## Architecture Decisions
- **Layout:** Switch to a collapsible **left sidebar** (mirrors the admin `AppSidebar` pattern) + a portal `Topbar` with notifications bell + avatar dropdown. Replaces the current top-nav.
- **Notifications:** Computed **on-the-fly** from live data (overdue dues, upcoming meetings within 7 days, recently resolved requests) — no new DB table, no migration.
- **Charts:** Lightweight inline SVG sparkline/bars for savings trend — no new dependencies.
- **Print statement:** `window.print()` with `@media print` CSS — no PDF library.
- **Auth/routing:** Unchanged. Reuses existing session guard + `proxy.ts`.

## 1. New Shared Components
**`components/PortalSidebar.tsx`** — Collapsible sidebar (reuses `AppSidebar` patterns: indigo gradient brand, section headers, active state, accordion-free flat nav, mobile Sheet). Nav groups:
- **Main:** Dashboard
- **Finance:** My Savings, My Loans, Apply for Loan
- **Account:** My Profile, My Requests, Settings

**`components/PortalTopbar.tsx`** — Topbar with: mobile hamburger, page context, theme toggle, **notifications bell** (dropdown showing computed notifications), avatar dropdown (name, member no, profile/settings/logout links).

**`components/PortalShell.tsx`** (optional client wrapper) — mirrors `DashboardClient.tsx` flex layout (sidebar + topbar + scrollable main).

---

## 2. Layout Rewrite — `app/portal/layout.tsx`
Convert from client top-nav to a **server component** that fetches computed notifications, then renders `<PortalShell>` (client) with `<PortalSidebar>` + `<PortalTopbar>` + children. Mobile sheet nav preserved.

---

## 3. Enhanced Dashboard — `app/portal/page.tsx`
Polished professional layout:
- **Welcome header** with member avatar, name, member no, membership-since date, KYC/status badges.
- **4 financial stat cards** (Current Balance, Total Deposited, Late Fines, Net Due) — refined with trend icons and accent colors.
- **Savings trend mini-chart** (SVG sparkline of last 6 months deposits).
- **Active loans summary card** (outstanding, next due date) — links to `/portal/loans`.
- **Two-column lower section:** Upcoming Meetings (left) + Recent Transactions (right, condensed).

---

## 4. New: My Loans — `app/portal/loans/page.tsx`
Lists all the member's loans (`prisma.loan.findMany({ where: { memberId } })`). Each row: loan no, product, principal, outstanding, status badge (reuses dashboard `STATUS_STYLES`), next due date. Empty state with CTA to "Apply for Loan". Stat cards: Total Borrowed, Total Outstanding, Active Loans.

## 5. New: Loan Detail — `app/portal/loans/[id]/page.tsx`
Full loan view: header (loan no, status, product, dates), terms (principal, rate, type, tenure, installment amount), running balances (paid/outstanding), **repayment schedule table** (installment no, due date, principal, interest, amount, status badge, paid date), and **repayment history**. Read-only (members can't record their own repayments — admin-only).

## 6. New: Apply for Loan — `app/portal/loans/apply/page.tsx` + `LoanApplyForm.tsx`
- Fetches active `LoanProduct` list server-side.
- Client form: select product (auto-fills rate/tenure/limits), amount, tenure, purpose, optional guarantors.
- **Live schedule preview** computed client-side using `generateSchedule` from `lib/loanSchedule.ts` (installment amount, total interest, total payable, first due date).
- Submits via new `applyMemberLoan` server action (wraps the existing `buildLoan(..., "PENDING")` logic from `app/actions/loan.ts`).

---

## 7. New: My Requests — `app/portal/requests/page.tsx`
Unified tracker the member currently lacks. Tabs: **Withdrawals | Profile Updates | Closing**. Pulls from `MemberRequest` (type WITHDRAWAL/CLOSING) and `ProfileUpdateRequest`, shows status badges (PENDING/APPROVED/REJECTED), submitted date, amount/reason/diff summary, and admin notes. Solves the "submitted and lost" problem.

---

## 8. My Savings Enhancements — `app/portal/savings/`
- Add **"Download / Print Statement"** button → opens print view (`@media print` CSS hides chrome, shows a clean statement header with org name, member info, and full transaction table with running balance column).
- Add a **running balance column** to the transaction history table.
- Polish stat cards and the withdrawal-requests panel (`MySavingsClient.tsx`).

---

## 9. Profile Polish + Photo Upload — `app/portal/profile/`
- **`ProfileEditDialog.tsx`:** Add **profile photo upload** (reuses `lib/cloudinary.ts → uploadImage`). Uploads to Cloudinary, then creates a `ProfileUpdateRequest` with `payload: { photoUrl }` for admin approval (per your selection). Shows "Photo pending approval" badge on the avatar.
- Refine the profile page visual hierarchy (cleaner section cards, consistent header treatment, better nominee cards). Keep all existing data fields/sections.

## 10. Companion: Admin approval handles `photoUrl`
Small addition to the existing admin profile-request approval action so that when a request's payload contains `photoUrl`, it's applied to `member.photoUrl`. (I'll locate and minimally extend `approveProfileUpdateRequest`.) Keeps the photo-approval flow end-to-end functional.

---

## 11. Settings Polish — `app/portal/settings/`
Refine the change-password card and danger-zone card visuals to match the new design language. No logic changes.

---

## 12. Server Actions — `app/actions/portal.ts` (extend)
Add:
- `applyMemberLoan(memberId, formData)` — guards the member is ACTIVE, delegates to a shared `buildLoan`-style helper producing a `PENDING` loan + schedule (reusing `generateSchedule`), then `revalidatePath("/portal/loans")`.
- `submitProfilePhotoRequest(memberId, file)` — uploads via Cloudinary, creates `ProfileUpdateRequest { payload: { photoUrl } }`.
- `getMemberNotifications(memberId)` — helper returning computed notifications (upcoming meetings, overdue dues, recent request resolutions).

- No member-side repayment recording (admin-only by design).
