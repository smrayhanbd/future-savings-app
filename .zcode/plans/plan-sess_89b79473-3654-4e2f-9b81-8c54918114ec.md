# Enterprise Loan Management System ‚Äî Implementation Plan

## Overview
Replace the current `ComingSoon` placeholder at `/dashboard/loans` with a full enterprise-grade Loan Management module. Multi-model interest (Flat default + Reducing Balance + Manual schedule), full lifecycle (Applications ‚Üí Approval ‚Üí Disbursement ‚Üí Repayment ‚Üí Close/Write-off) **plus** a direct-disburse fast path, dedicated `LoanRepayment` table mirrored to the existing `Savings` ledger (`type=LOAN_PAYMENT`).

This matches every existing convention: Prisma + Server Actions, `revalidatePath`/`redirect`, base-ui `Select` with `onValueChange`, premium dark-aware card aesthetic, `force-dynamic` server pages, Sonner toasts.

## 2. Schedule Engine ‚Äî `lib/loanSchedule.ts` (new)
Pure functions, fully unit-testable. Mirror style of existing `lib/dueCalculator.ts`.

- `addInterval(date, freq, n)` ‚Äî handles DAILY/WEEKLY(+7)/FORTNIGHTLY(+14)/MONTHLY(+n months, clamped)/QUARTERLY/YEARLY.
- `generateFlatSchedule(principal, annualRate, freq, installments, disburseDate, grace)` ‚Äî interest = principal √ó rate √ó tenure-years; EMI = (principal+interest)/installments; each row splits evenly.
- `generateReducingSchedule(principal, annualRate, freq, installments, disburseDate, grace)` ‚Äî standard amortization: per-period rate = annual/(freq-per-year); EMI formula; per-row interest=balance√órate, principal=EMI‚àíinterest.
- `generateManualSchedule(rows[])` ‚Äî pass-through for admin-defined installments.
- Return normalized `{installmentNo, dueDate, principal, interest, installmentAmount, balanceAfter}` rows (no Prisma types) so the same engine previews schedules in the UI **before** save.

## 3. Server Actions ‚Äî `app/actions/loan.ts` (new)
Pattern from `finance.ts` / `accounts.ts`: `"use server"`, prisma import, `revalidatePath`/`redirect`, FormData for form-posted actions, plain args for button-driven ones.

**Loan Products**
- `createLoanProduct(formData)`, `updateLoanProduct(id, data)`, `toggleLoanProductStatus(id, isActive)`, `deleteLoanProduct(id)` (guarded if loans exist).

**Loan lifecycle**
- `applyLoan(formData)` ‚Äî creates Loan `PENDING` + auto-generates `LoanSchedule` rows from engine + stores guarantors. Validates member ACTIVE.
- `quickDisburseLoan(formData)` ‚Äî direct-entry fast path: creates Loan already at `DISBURSED`, sets `disbursedDate`/`expectedCloseDate`/`outstandingBalance`, generates schedule.
- `approveLoan(id)`, `rejectLoan(id, reason)` ‚Äî PENDING ‚Üí APPROVED/REJECTED.
- `disburseLoan(id, method, date)` ‚Äî APPROVED ‚Üí DISBURSED, sets balances & nextDueDate.
- `recordRepayment(loanId, {principal, interest, fine, method, ref, date})` ‚Äî the core:
  1. Transaction: insert `LoanRepayment`, **mirror to `Savings`** (`type=LOAN_PAYMENT`, same amount, member), link via `savingsMirrorId`.
  2. Apply against schedule: oldest PENDING installment ‚Üí mark PAID/PARTIAL, advance `nextDueDate`.
  3. Update Loan running balances (`principalPaid`, `interestPaid`, `finePaid`, `outstandingBalance`).
  4. Auto-close (`status=REPAID` ‚Üí `CLOSED`) when outstanding ‚â§ 0.
- `waiveInterest(loanId, installmentNo)`, `rescheduleLoan(loanId, newTerms)` (guarded by product flags), `writeOffLoan(id)`, `earlySettleLoan(id, date)` (waiver of future interest per product flag).
